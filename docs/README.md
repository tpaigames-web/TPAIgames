# 🧱 项目结构规范（Day 1 起强制执行）

## 📁 目录结构（不可随意更改）

/scenes /core /map /enemy /tower /ui

/scripts /core /enemy /tower /manager

/assets /sprites /tower /enemy /ui /background /audio

新增目录必须提前讨论。

------------------------------------------------------------------------

# 📛 命名规范（强制）

### 1️⃣ 所有文件名必须英文

❌ 稻草人.tscn\
✅ Scarecrow.tscn

❌ 金币管理器.gd\
✅ GoldManager.gd

------------------------------------------------------------------------

### 2️⃣ 统一 PascalCase 命名

场景 / 类名： Scarecrow EnemyBase WaveManager GameManager

变量： max_hp attack_speed move_speed

常量： MAX_WORKERS BASE_GOLD_REWARD

------------------------------------------------------------------------

# ⚙️ 代码规则（必须遵守）

## 1️⃣ 所有数值必须可数据化

禁止写死数值：

hp = 100

必须使用：

@export var max_hp : int = 100

原因：

-   后期有跨关卡倍率
-   有无限模式
-   要做平衡性调整

------------------------------------------------------------------------

## 2️⃣ 禁止硬编码路径

使用 preload 或统一资源管理方式。

------------------------------------------------------------------------

## 3️⃣ 所有怪物和子弹必须未来支持对象池

Day 1 不实现对象池\
但架构必须可扩展

------------------------------------------------------------------------

## 4️⃣ 逻辑分层规则

GameManager：游戏状态、胜负判断\
WaveManager：波次生成\
EnemyBase：HP、护甲、死亡逻辑\
TowerBase：攻击与升级逻辑

禁止混乱调用。

------------------------------------------------------------------------
