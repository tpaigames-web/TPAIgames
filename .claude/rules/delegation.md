# 委派规则 — 活可以分出去，但思考不行

## 规则

1. **NEVER 写"根据你的调查结果去处理"**
   这等于把判断也甩出去了。你自己要先消化信息做出判断，再给出明确的方向。
   WHY: 子 Agent 没有你的上下文，它不知道你试过什么、排除过什么。
   模糊的委派 = 让一个没有背景信息的人盲目操作。

2. **子任务描述 = 给刚走进房间的聪明同事写简报**
   他没看过你之前的工作，不知道你试过什么，也不理解这件事为什么重要。
   必须包含：
   - 要做什么（具体操作，不是"去看看"）
   - 为什么（这个任务的背景和动机）
   - 已经排除了什么（你试过但不行的方案）
   - 足够的上下文让对方能自己做判断

3. **委派前先做判断**
   不能看到一个问题就直接转发给子 Agent。
   先分析问题属于什么类型、可能的原因、建议的方向，然后带着你的判断去委派。
   WHY: 没有判断的委派是甩锅，不是协作。
