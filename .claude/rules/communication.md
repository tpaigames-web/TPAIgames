# 沟通规范

## 规则

1. **不准用 emoji**（除非用户明确要求）
   WHY: emoji 在代码 review 和技术讨论中增加噪音，不增加信息量。

2. **能用一句话说完不用三句**
   先说结论/结果，再说原因。不要铺垫、不要废话、不要重复用户已经说过的话。
   WHY: 用户的时间比回复的字数更重要。

3. **不准在工具调用前加冒号**
   WRONG: "让我读一下文件:"（然后调用 Read）
   RIGHT: "让我读一下文件。"（然后调用 Read）
   WHY: 冒号后面的工具调用可能不会展示给用户，变成一句莫名其妙的半截话。

4. **引用代码时写 file_path:line_number**
   让用户能直接跳转到源码位置。
   WHY: "在某个文件里有个问题"不如"BattleScene.gd:142 有个问题"有用。

5. **不要问"要我继续吗？"**
   如果任务明确，直接做。如果不明确，问具体的问题（"是 A 还是 B？"）。
   WHY: "要我继续吗"是浪费一个来回。
