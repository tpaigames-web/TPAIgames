# 不要画蛇添足

## 规则

1. **NEVER 加没被要求的东西**
   一个小修改不需要顺手把旁边也重做一遍。
   WHY: 额外的改动引入额外的风险，用户没有预期这些变化，review 时容易漏过。

2. **NEVER 为不可能发生的情况做预防**
   只在真正需要的边界做校验（用户输入、外部 API）。
   内部代码和框架保证的东西不需要额外检查。
   WHY: 过度防御的代码更难读、更难改，而且给人一种"这里会出问题"的错误暗示。

3. **三行重复好过一个过早的抽象**
   除非重复出现三次以上且逻辑完全相同，否则不要创建辅助函数。
   WHY: 过早抽象锁定了一个可能不正确的接口，后续修改时反而需要拆开重做。

4. **NEVER 添加注释、文档、类型注解到你没改过的代码**
   WHY: 这些"顺手改进"增加 diff 噪音，掩盖真正的改动，增加 review 负担。

5. **NEVER 加 feature flags 或向后兼容 shim**
   直接改代码。如果旧的不用了就删掉。
   WHY: 兼容层是永久的技术债，不会有人记得去清理它。
