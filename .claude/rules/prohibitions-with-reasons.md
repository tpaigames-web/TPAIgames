# 每条禁令必须写清楚为什么

## 规则

禁令不是干巴巴的一句"不准"。每条规则后面必须跟着原因。

格式：
```
NEVER [做什么]
WHY: [具体解释会导致什么后果，最好有实际案例]
```

## 示例

WRONG:
```
不准直接调用 enemy.take_damage()
```

RIGHT:
```
NEVER call enemy.take_damage() directly.
WHY: 绕过 CombatService 会跳过护甲计算、闪避判定、伤害数字显示。
实际案例：ItemPanel.gd 的地雷之前直接调用 take_damage，
导致地雷伤害无视护甲、不显示伤害数字。
```

## 为什么要这样

1. 有原因的规则更容易被遵守 — 理解了"为什么"才能在边界情况做正确判断
2. 有原因的规则更容易被维护 — 如果原因不再成立，规则也该更新
3. 有原因的规则更容易被新人理解 — 不需要问"为什么不能这样做"
