# 硬规则：先读后改

## 规则

**CRITICAL: 必须先读过一个文件的内容，才能去编辑它。**

没读就改，系统直接报错，不让你操作。

WHY: 这条防的是 AI 最常见的问题 — 凭记忆或者幻觉去修改东西，
改出来的跟实际内容对不上。上次就因为两行残留的 `dlg` 变量引用
（BattleScene.gd:1140-1141），导致整个脚本 Parse Error，
所有功能（炮台卡片、设置按钮、强化面板）全部失效。

## 具体要求

1. 编辑任何 .gd/.tscn/.tres 文件前，必须先用 Read 工具读取该文件
2. 如果文件超过 200 行，至少读取要修改的部分 ± 20 行上下文
3. 读完后确认要修改的 old_string 在文件中确实存在且唯一
4. 如果不确定，用 Grep 搜索确认
